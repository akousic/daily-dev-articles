## Summary

Uber engineers have re‑architected the company’s data lake ingestion platform, moving from scheduled batch jobs to a streaming‑first system. The platform, IngestionNext, processes event streams continuously, reducing ingestion latency from hours to minutes and enabling faster availability for analytics and machine learning workloads. Previously, ingestion pipelines relied on Apache Spark and ran as scheduled batch jobs.

## Key Takeaways

- While capable of large-scale processing, batch pipelines delayed data availability for analytics and experiments.
- As noted by Kai Waehner, Global Field CTO in a LinkedIn post: This move is all about treating data freshness as a key dimension of data quality.
- IngestionNext introduces a streaming-first pipeline that continuously processes event streams before committing them to the data lake.
