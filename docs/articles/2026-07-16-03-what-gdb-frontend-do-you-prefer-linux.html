What gdb frontend do you prefer (linux)

LobstersRank #3 · RSS curated source · 2026-07-16 08:12 UTC

Open original article ↗

Summary

I like QtCreator's debugger, it's nicely laid out. You don't have to create a QtCreator project to debug with it. You can either start an executable via a dialog or connect to a running program by PID.

Key Takeaways

  • Would highly recommend pwndbg.
  • Don’t be fooled by the lack of a GUI app, as this presents much more (useful) information than stock gdb while remaining a TUI.
  • Yes, many of the features are catered to binary exploitation specifically, but as it turns out things like being able to inspect the glibc heap are also useful for tracking down strange allocation patterns.
